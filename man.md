# Professor Manual (Attendo / UNYT)

## 1. Purpose
This manual explains how professors manage attendance sessions, student accounts, and excuse requests in the platform.

## 2. Login
1. Open the Professor Portal.
2. Sign in with your professor username and password.
3. If login fails, confirm that you are using the professor portal and correct credentials.

## 3. Starting Attendance
1. Open your course session from the dashboard.
2. Start a new round.
3. Show the QR code to students.
4. Optionally share the active **Class ID** for manual testing/check-in when needed.

## 4. During Class
1. Monitor live attendees.
2. Watch scan status and total attendance.
3. Use Close Round when attendance window ends.
4. Use End Session when class is complete.

## 5. Student Accounts
1. Create student accounts manually or via roster import.
2. Share credentials with students (username + temporary password).
3. Ask students to change password at first login.
4. If students forget password, reset from professor tools and resend temporary credentials.

## 6. Excuse and Day-Off Handling
1. Students who miss class must submit an excuse request on the platform.
2. Students should also notify you via email.
3. Review pending excuses and approve/reject with notes.

## 7. Best Practices
1. Start rounds only when ready to scan.
2. Keep QR visible and stable for students.
3. Remind students to use their own accounts and devices.
4. Export attendance reports regularly.

## 8. Troubleshooting
1. If scans fail, confirm session and round are active.
2. If QR expires too quickly, increase `QR_TOKEN_TTL_SECONDS` in backend environment.
3. If login/API fails in production, verify frontend/backend environment variables and CORS settings.
